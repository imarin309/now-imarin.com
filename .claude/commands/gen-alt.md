MDXファイルの画像に適切なalt属性を生成してください。

## 対象ファイル

`$ARGUMENTS` で指定されたMDXファイルを読み込んでください。
引数が空の場合は、対象ファイルのパスを聞いてください。

## 手順

1. 対象MDXファイルを読み込む
2. フロントマターから記事タイトル・タグ・文脈を把握する
3. 以下のパターンで alt が **空文字・未設定・`""`** になっている画像を探す
   - `ImageGallery` の `images` 配列内の `{ src: "...", alt: "" }` または `alt` キーなし
   - Markdown記法 `![](...)`（角括弧が空）
   - `<img src="..." alt="" />` または `alt` 属性なし
4. 各画像について alt を生成する（後述のルール参照）
5. ファイルを直接編集して alt を埋め込む

## alt生成ルール

- **フォーマット**: `{製品名・キャラクター名}_{工程・部位・状態}`
  - 例: `HG_ガンダム_下地塗装`、`figma_カナヲ_リペイント後正面`
- **ヒントにするもの（優先順）**:
  1. `caption` の内容（同じ `{}` 内にある場合）
  2. 画像URLのファイル名（`-` や `_` で区切られた単語）
  3. 前後の本文テキスト
  4. 記事タイトル・タグ
- **禁止**: キーワード詰め込み、「画像」「写真」などの汎用ワード
- **言語**: 日本語（英語の製品名・型番はそのまま使ってよい）

## 出力

ファイルを編集した後、変更した画像の一覧を以下の形式で表示してください:

```
## 生成したalt一覧

| # | src（ファイル名のみ） | 生成したalt |
|---|---|---|
| 1 | hg-gundom-package.webp | HG_ガンダムグラスフェザー装備_パッケージ |
```

altが空の画像が存在しなかった場合はその旨を伝えてください。
