ブログ記事を英語に翻訳し、多言語対応（i18n）してください。

## 対象ファイル

`$ARGUMENTS` で指定されたMDXファイルを読み込んでください。
引数が空の場合は、IDEで開いているファイルを候補として提示し、なければ対象のファイルパスを聞いてください。

対象は `content/posts/<slug>.mdx`（日本語オリジナル）を想定します。

## 手順

### 1. 既存の翻訳パターンを確認する

`content/posts/en/` 配下に既存の翻訳済み記事があれば1つ読み、フロントマターの書き方・文体・訳し方のトーンを揃える。

### 2. 固有名詞の公式英語名を確認する

記事内でゲーム名・作品名・商品名などの固有名詞が使われている場合、思い込みで直訳せず `WebSearch` で公式サイトや公式Xアカウントを検索し、正式な英語表記を確認してから使う（例:「星の翼」→ 公式サイトの英語版で "Starward" と確認する）。
確認できた場合は、本文中で紹介している公式サイトURLも英語版のURL（存在すれば）に差し替える。

### 3. 翻訳して新規ファイルを作成する

`content/posts/en/<同じslug>.mdx` を新規作成する（ファイル名・slugは日本語版と同一にする）。

- **フロントマター**
  - `title` / `description`: 英語に翻訳する。`description` は検索ユーザー向けに自然な英語にする
  - `date` / `category` / `coverImage` / `tags` / `noindex`: 日本語版の値をそのまま引き継ぐ（翻訳しない）
- **本文**
  - 見出し構成・段落構成は日本語版と一致させる
  - `:large[]` `:red[]` `:blue[]` `:pink[]` などのremark-directive記法はそのまま維持し、中身のテキストだけ翻訳する
  - 絵文字・顔文字（( ・∇・) など）はそのまま残す
  - `<RakutenCard>` / `<RakutenCardImage>` コンポーネントは翻訳しない（`href` `imgSrc` `title` `price` は日本語版のまま、日本の読者向けアフィリエイトのため）
  - 表（テーブル）はヘッダー・内容ともに翻訳するが、数値やキャラクター名などの固有名詞は原文の表記に合わせる
  - URLは `[テキスト](URL)` 形式かベアURLで記載し、`<https://...>` 形式のautolinkは使わない（MDXでJSXとして誤解釈されエラーになるため）

### 4. Lintを通す

作成したファイルに対して以下を実行し、エラー・警告がなくなるまで修正する。

```bash
npx prettier --write content/posts/en/<slug>.mdx
npx markdownlint-cli2 "content/posts/en/<slug>.mdx"
```

特にテーブルは手動で書くと列幅がPrettierの整形結果とずれやすいので、`prettier --write` 後の内容を正とする。

## 出力

- 作成したファイルパスを報告する
- 固有名詞の英語表記をWebSearchで確認した場合は、その根拠（参照した公式ソース）を簡潔に示す
- Lint結果（prettier / markdownlint）が通ったことを報告する
- ユーザーの指示がない限りコミットは行わない

## 注意事項

- 既存の日本語記事の内容・構成は変更しない（新規ファイルの追加のみ）
- 1回の実行で1記事のみ翻訳する
